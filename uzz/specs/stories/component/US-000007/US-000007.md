# User Stories for Domain: Component

## [US-000007] Persist WebView session cookies across restarts
**As a** user,
**I want** the WebView session cookies to persist across application restarts,
**So that** end users remain logged into their chosen LLM chat provider between sessions.

### Technical Context & Constraints
- **Target System:** JavaFX desktop applications
- **Toolchain / Rules:** JavaFX, Maven, JUnit 5, TestFX, AssertJ, headless testing, TDD workflow, snake_case test names, use `then()` for BDD assertions

### Acceptance Criteria
```gherkin
Scenario: Session cookie set on first run survives a second run with default storage
  Given the component is instantiated without providing a specific session storage directory
  And the component loads a fake page that sets a cookie
  When a new component instance is created without providing a specific session storage directory
  And the new instance loads the same fake page
  Then the fake page receives the previously set cookie

Scenario: Session cookie set with a custom storage directory survives a second run using the same directory
  Given the component is instantiated with a specific session storage directory
  And the component loads a fake page that sets a cookie
  When a new component instance is created with the same session storage directory
  And the new instance loads the same fake page
  Then the fake page receives the previously set cookie

Scenario: Session cookies do not leak between different custom storage directories
  Given the component is instantiated with one session storage directory
  And the component loads a fake page that sets a cookie
  When a new component instance is created with a different session storage directory
  And the new instance loads the same fake page
  Then the fake page does not receive the previously set cookie

Scenario: Reject Invalid custom storage directory
  Given the component is instantiated with a non-writable custom session storage directory
  When the component is created
  Then an IllegalArgumentException is thrown
```
