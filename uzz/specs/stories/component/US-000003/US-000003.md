# User Stories for Domain: Component

## [US-000003] Load LLM entries from configuration
**As a** JavaFX application developer,
**I want** to provide LLM names and URLs through configuration,
**So that** the component can be reused without hard-coded provider data.

### Technical Context & Constraints
- **Target System:** JavaFX desktop applications
- **Toolchain / Rules:** JavaFX, Maven, JUnit 5, TestFX, AssertJ, headless testing, TDD workflow, snake_case test names, use `then()` for BDD assertions

### Acceptance Criteria
```gherkin
Scenario: Component reads LLM entries from a configuration object
  Given a configuration object contains multiple LLM names and URLs
  When the component is initialized
  Then the selector is populated with the configured entries

Scenario: Component reads LLM entries from a configuration function
  Given a configuration function returns multiple LLM names and URLs
  When the component is initialized
  Then the selector is populated with the returned entries
```
