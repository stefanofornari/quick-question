# User Stories for Domain: Demo

## [US-000005] Dynamically add WebChat entries to the dropdown
**As a** user,
**I want** newly added WebChat entries to appear dynamically in the dropdown,
**So that** I can verify the demo updates its selectable LLM list without restarting the application.

### Technical Context & Constraints
- **Target System:** JavaFX desktop application
- **Toolchain / Rules:** JavaFX, CommonsFX, AtlantaFX, Maven, JUnit 5, TestFX, AssertJ, headless testing, TDD workflow, snake_case test names, use `then()` for BDD assertions

### Acceptance Criteria
```gherkin
Scenario: Existing entries appear in the listbox
  Given the demo application is launched
  When the main window is displayed
  Then the listbox contains the configured WebChat entries

Scenario: Newly added WebChat entries appear in the dropdown
  Given the demo application is launched
  And the user enters a new LLM name and URL in the configuration UI
  When the user applies the changes
  Then the dropdown contains the newly added WebChat entry
  And the dropdown remains usable without restarting the application

Scenario: The dropdown reflects the current configuration after multiple additions
  Given the demo application is launched
  When the user adds multiple WebChat entries through the configuration UI
  And the user applies the changes
  Then the dropdown contains all configured WebChat entries in the expected order
```
