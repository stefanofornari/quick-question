# User Stories for Domain: Demo

## [US-000009] Keep browser session across program runs
**As a** user,
**I want** that when I reopen the app, the chats session information are maintained
           like I was using a standard browser
**So that** I do not have to login again, I see the history, etc...

### Technical Context & Constraints
- **Cookie management:** the only way to achieve this behaviour is by providing
  a custom CookieHandler to java.net.CookieManager. Note that this affect all
  HTTP requests done through the java.net package.
- **Documentations:** the use of CookieManager shall be documented in a dedicated
  section of the project README.

### Acceptance Criteria

```gherkin
Scenario: Cookies are persisted between application restarts
    Given the user has authenticated with the chat service -> a session cookie has been issued
    When the application is closed and reopened
    Then the user is still authenticated
    And the session cookie has the values set in the session
Scenario: Cookie management is documented
    Given the project README.md is available
    When opened
    Then it explains that a custom CookieHandler is used
    And it describes how cookies are persisted across program runs
```
